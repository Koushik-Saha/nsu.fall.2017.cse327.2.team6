3.0 Detailed Description of Components



3.1 Component Template Description



Identification

The unique name for the component and the location of the component in the system.

Type

A module, a subprogram, a form, a data file, a control procedure, a class, etc

Purpose

Function and performance requirements implemented by the design component, including derived requirements. Derived requirements are not explicitly stated in the SRS - but are implied or adjunct to formally stated SDS requirements.

Subordinates

The internal structure of the component, the constituents of the component, and the functional requirements satisfied by each part.

Dependencies

How the component’s function and performance relate to other components. How this component is used by other components. The other components that use this component. Interaction details such as timing, interaction conditions (such as order of execution and data sharing), and responsibility for creation, duplication, use, storage, and elimination of components.

Interfaces

Detailed description of all external or internal interfaces as well as of any mechanism for communicating through messages, parameters, or common data areas. All error messages and error codes should be identified. All screen formats, interactive messages, and other user interface components (originally defined in the SRS) should be given here.

Resources

A complete description of all resources (hardware or software) external to the component but required to carry out its functions.

Processing

A full description of the functions presented in the Function subsection. Pseudocode can be used to document algorithms, equations, and logic.

Data

For the data internal to the component, describes the representation method, initial values, use, semantics, and format.









3.2 Description of Client



Identification

ClientMenu

Type

Class/Form

Purpose

The Client menu assures that only buyers/clients can access the system and the ability to create a unique user name for himself

Subordinates

This screen contains links to the following screens:

  • Login Screen

  • Main Menu Screen

  • New User Account Screen

Dependencies

The following screen links to this screen:

  • Main Menu Screen

  • Login screen

Interfaces

The links are contained in the top half of the screen. The screen is designed to be easy to view using the resolution standard on the PDA.

Resources

Database Access Requirements: access to the clients found in the appropriate database tables. This access is used to create a new user and check to make sure not the same as another user.

Processing

The only type of processing required is inputting information into the text boxes and navigating to other forms using links in the top quarter of the screen. Each link directs the user to a different screen that corresponds to the link that the user selects.

Data

The data for the main menu contains the fields “view cows”, “Search”, “special requirements”, “Appoint A butcher”, “Appoint A Doctor” and the login screen is the username and password entered by the user. It is validated with a query against the database. The data supplied by the system are fields the new user must enter. The data given by the user is the appropriate information needed to fill in the given fields. This data once determined valid, by checking to make sure the user does not already exist, is saved in the database.
